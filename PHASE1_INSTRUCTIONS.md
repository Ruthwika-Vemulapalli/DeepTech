# Phase 1 Instructions (DeepTech Hackathon 2026)

## File naming + slide limit
- **File name:** `TeamName_Phase1.pdf`
- **Max slides:** 6–7 including title (keep to this limit)

## Slide flow (6–7 slides)

1. **Title + Team + PS code + Phase tag**
2. **Problem statement** (your understanding, not copy-paste)
3. **Idea summary + approach**
4. **Proposed solution** (architecture overview) **+ Dataset plan + classes + split**
5. **Model baseline + metrics + model size** (Technology & Feasibility / Methodology)
6. **Artifacts & Links** (GitHub + dataset + ONNX) + references if space
7. (Optional) **Research & References** if not merged into slide 6

## Must-have content

### Dataset Plan & Class Design (slide 4 or 5)
- Total images planned/current: ___
- No. of classes: ___ (Min 8: 6 defect + Clean + Other)
- Class list: Defect_1…Defect_6 + Clean + Other
- Class balance plan: (min per class)
- Train/Val/Test split: ___ / ___ / ___
- Image type: Grayscale preferred (mention)
- Labeling method/source: manual / public dataset / generated

### Baseline Model & Results (Phase 1) (slide 5)
- Architecture, training approach, input size, model size, framework
- Accuracy, Precision/Recall, Confusion Matrix (image or table)

### Artifacts & Links (slide 6)
- GitHub Repository (mandatory)
- Dataset ZIP link (Drive/Kaggle/HF)
- ONNX model link
- Results report link (optional)
- Video: optional for Phase 1

### Research (slide 6 or 7)
- 2–3 references max
- Dataset source citation
- NXP/eIQ docs link if referred
